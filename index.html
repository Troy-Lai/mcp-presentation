💡 解决方案

MCP = AI 世界的 USB-C 标准

就像 USB-C 让所有设备用同一个接口充电和传数据，MCP 让所有 AI 用同一种方式连接工具和数据源

Model Context Protocol (MCP) 是一个开放协议，由 Anthropic 于 2024 年 11 月发布。

它定义了 AI 与外部工具之间的标准化通信方式——基于 JSON-RPC 2.0，通过 stdio 或 HTTP 流式传输（2026 新版支持）。

核心理念：写一次 MCP Server，所有 AI 客户端都能用。
Claude Code、Cursor、Gemini CLI、OpenAI Agents SDK 等全部支持。

🧬 协议核心

MCP 的三大原语：Tools / Resources / Prompts

整个协议围绕这三个概念展开——简单但强大，覆盖 AI 与外部交互的所有模式

🔧

Tools（工具）

AI 可调用的函数——每个工具定义名称、描述、输入/输出 JSON Schema。AI 自己决定何时调用什么工具，像人一样"使用工具"。

"search_weather"
→ 输入：{ city: "上海" }
← 输出：{ temp: 26, condition: "晴" }
📂

Resources（资源）

AI 可读取的结构化数据——文件内容、数据库 schema、API 文档。资源可动态更新，AI 按需拉取，像人一样"查阅资料"。

"project://src/schema.ts"
→ 返回文件内容
"db://users/table_info"
→ 返回表结构
💬

Prompts（提示模板）

预定义的对话模板——引导 AI 如何使用工具、格式化输出。包含参数，动态生成上下文，像给 AI 一份"标准作业流程"。

"summarize_recent"
→ 模板："总结最近 {days} 天
的 {topic} 相关内容"
← AI 按模板整理输出

⚡ 快速上手

3 步给 Claude Code 装上 MCP

以接入天气查询和文件系统为例，30 秒完成配置

📋 Step 1: 添加 MCP Server
终端
# 添加天气查询 MCP Server claude mcp add weather -- npx -y @h1deya/mcp-server-weather # 添加文件系统 MCP Server claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /path/to/allowed # 查看已添加的 Server claude mcp list
⚙️ Step 2: 自动生成的配置
.claude/mcp.json
{ "mcpServers": { "weather": { "command": "npx", "args": ["-y", "@h1deya/mcp-server-weather"] }, "filesystem": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path"] } } }
✅ Step 3: 直接用！ 重启 Claude Code，AI 自动发现新工具。试试说 "帮我查上海天气" 或 "读一下 src/ 目录下的所有 .ts 文件"。
💡 经验分享

实际搭建 MCP 的踩坑和心得

我折腾过程中学到的——帮你少走弯路

✅

从官方 Server 开始

先装 @modelcontextprotocol 官方仓库的 Server（filesystem、github、postgres），验证链路跑通再自己写。

⚠️

注意文件系统权限

Filesystem MCP 启动时指定的目录就是 AI 能访问的全部范围。限定到项目目录，别把根目录给它。

💡

工具描述决定 AI 表现

MCP Server 的 tool description 写得好，AI 才懂得何时调用。要包含：功能、适用场景、参数含义。

🔧

Python 用户用 FastMCP

写自定义 MCP Server 推荐 FastMCP——5 行代码定义一个 tool，比自己撸 JSON-RPC 省心太多。

🐛

调试技巧：看日志

MCP 跑在 stdio 上，调试不方便。Claude Code 加 --debug 参数可以看到完整的 MCP 通信过程。

🚀

组合多个 MCP 才是精髓

单个 Server 能力有限，把 filesystem + github + postgres 连起来，AI 就能从需求到代码走完全链路。
